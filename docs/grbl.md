# GRBL 调试记录

## 硬件连接

- **控制板**：ESP32 GRBL 六轴，CH340 USB 串口
- **串口**：COM18（CH340），波特率 115200
- **驱动模块**：TMC2209 ×2，插 X、Y 座；方向与 A4988 **相反**（芯片面朝 ESP32 侧，板上标注"2209 则相反，反插会烧"）
- **限位开关**：当前实物接线为带指示灯的小板方案。小板控制器侧丝印为 `S/G/V`，开关侧丝印为 `C/NO/NC`
- **限位接线**：控制板 `X/Y` 限位信号接小板 `S`，控制板 `GND` 接小板 `G`，`5V` 接小板 `V`
- **限位状态**：实测松开时常见 `Pn:XYZ`；按下 `X` 时变为 `Pn:YZ`，按下 `Y` 时变为 `Pn:XZ`

## GRBL 参数

> **重要**：CH340 开串口会触发 ESP32 复位。调试时避免多个进程并发打开同一个串口，否则可能出现复位时序和命令流打架，导致运动异常。

### 已验证（必须每次连接后发送）

| 参数 | 值 | 说明 |
|------|-----|------|
| `$3` | `2` | 方向反转掩码：反转 Y 轴（bit1） |
| `$5` | `0` | 当前 `S/G/V` 小板接法下按现场实测使用 `0`；早期原始 NC 直连接法曾用过 `3` |

### 当前工作参数（2026-04-04 已复测）

| 参数 | 值 | 说明 |
|------|-----|------|
| `$100` | `80.208` | X 步进/mm；按真实全行程 `510mm` 标定，并复测通过 |
| `$101` | `80.110` | Y 步进/mm；按真实全行程 `476mm` 标定，并复测通过 |
| `$120` | `250` | X 加速度 mm/s²；由控制程序连接后写入 |
| `$121` | `250` | Y 加速度 mm/s²；由控制程序连接后写入 |

## 调试日志

### 2026-03-17 XY 基础运动验证

- XY 轴电机均可正常转动，TMC2209 工作正常
- Y 轴出厂方向相反，`$3=2` 修正后验证通过
- 早期原始 NC 直连接法需 `$5=3` 反转极性
- 增量模式 `G91` + `G1 X15 / Y15 F500` 测试通过，两轴方向正确、回位准确

### 2026-04-04 坐标方向复核

- 通过 NUC SSH 向 `/dev/ttyUSB0` 发送增量点动指令 `G91`、`G1 X10 F500`、`G1 Y10 F500` 进行复核
- XY 限位改为带小板接法：控制板限位信号→`S`，控制板 `GND`→`G`，`5V`→`V`
- 小板供电后状态正常；实测松开时常见 `Pn:XYZ`，按 `X/Y` 时分别变为 `Pn:YZ` / `Pn:XZ`
- 在当前机械安装姿态与参数配置（`$3=2`、`$5=0`）下，`+X`、`+Y` 均为**远离对应限位块**的方向
- 发送 `G1 Y-10 F500`、`G1 X-10 F500` 后可回到原位

### 2026-04-04 写参后异常排查

- 现场出现过一次“写入 `$100` 后执行 `G91 G1 X1 Y1 F200`，X 持续朝限位方向跑”的异常
- 之后在同样参数、同样起点、同样指令格式下多次复现测试，`X+1`、`Y+1`、`X+1 Y+1` 均正常，未能复现确定性的参数问题
- 已定位到一个高风险点：当时存在两个进程并发访问 `/dev/ttyUSB0`。该板卡开串口会触发复位，并发访问可能打乱复位与命令时序
- 为避免再次出现同类问题，`tools/grbl_xy_calibrate.py` 已增加串口文件锁；同一时刻只允许一个该脚本实例占用串口

### 2026-04-04 XY 全行程标定结果

- 用户给定真实机械全行程：`X=510mm`、`Y=476mm`
- 从 `XY` 最大端自动回测到限位后，先得到 `X=548` 控制器单位、`Y=476` 控制器单位
- 反算并写入参数后，最终工作参数确认为：`$100=80.208`、`$101=80.110`
- 使用新参数再次从 `XY` 最大端自动复测，全行程结果为 `X=510.000`、`Y=476.000`
- 结论：当前 `X/Y` 步进参数已与机械实测全行程对齐，暂不需要继续校准
