# 幽灵棋盘技术规划

## 概述

中国象棋自动对弈棋盘，当前默认使用直接拖动的扁平棋片（引磁片上贴棋子贴纸）在棋盘表面滑动。

## 当前方案

- 棋盘顶板采用 `2mm` 单层透明板，棋盘层贴在上表面
- 当前不采用永磁体锁定，不再继续更换更强的同类吸盘式电磁铁
- 电磁铁最低点不再增加额外厚度，优先直接压缩磁距
- 当前控制方式采用“连续物理路径 + 终点释放补偿”，路径不限制在棋盘格线上
- 旧方案和已放弃路线统一归档到 [`docs/archive/abandoned-options.md`](archive/abandoned-options.md)

## 系统结构

```
USB 摄像头（UVC，俯拍）
        │
        ▼
Intel NUC7i5BNH（Linux / Ubuntu）
  ├─ 外部视觉仓库 `GhostVision`
  ├─ 象棋 AI
  └─ USB 串口 → G-code
        │
        ▼
ESP32 GRBL + TMC2209 ×2
  ├─ XY 运动
  └─ PWM → MOSFET → 电磁铁
```

## 机械与棋盘

- 顶板：`2mm` 透明亚克力或 PC
- 棋盘图案：当前使用备用棋盘层贴于上表面
- 棋面载体：直接使用 `Φ20×0.3mm` 引磁片作为被拖对象，上表面贴棋子图案贴纸
- 木质棋子：已退出当前默认工作形态，仅保留早期验证记录
- 电磁铁：当前已验证 `5W` / `8W` 吸盘式型号；两者都能驱动，但都不是最终长期方向
- 木板固定：当前做法为在桐木板上打通孔后，使用 `4` 颗带垫圈的螺钉穿过木板，并以约 `8cm` 的间距拧入铝型材槽内的 `T` 型螺母，从而把木板固定到 XY 框架
- 侧面按钮优先，避免上表面入镜和误触

## 运动与控制

### 电磁铁驱动

- GRBL 激光接口输出的是约 `0V/5V` PWM 逻辑电平
- 电磁铁通过外接 MOSFET 低边开关驱动，不直接挂在板载接口上
- 指令约定：
  - `M3 Sxxx`：电磁铁通电
  - `M5`：电磁铁断电
  - `$30`：PWM 上限，当前按 `1000` 使用

### 坐标与格距

- `y` 轴平行于楚河汉界
- `x` 轴垂直于楚河汉界
- 当前控制格距基于实测有效格区：
  - `x = 370 / 9 ≈ 41.11mm/格`
  - `y = 337 / 8 = 42.125mm/格`
- 说明：
  - `370mm` 对应主棋盘沿 `x` 方向的 `9` 格总长
  - `337mm` 对应主棋盘沿 `y` 方向的 `8` 格总长
  - 以上仅统计格子区域，不含棋盘纸外圈边距

### 拖动动作

- 当前默认拖动动作：
  - 电磁铁移动到棋子中心
  - `M3 Sxxx` 吸合
  - 沿连续物理路径拖动棋子
  - 按规划出的释放方向执行终点 `release_overshoot`
  - `M5` 断电
  - 电磁铁空载回到目标点中心
- 当前默认初值：
  - `release_overshoot = 15mm`
  - `magnet_exclusion_radius = 30mm`
  - `soft_clearance = 8mm`
- 当前默认速度：
  - 带棋拖动 `move_feed = 2400 mm/min`
  - 空载移动 `return_feed = 24000 mm/min`
- 中间路径点不做释放补偿；释放补偿只发生在最终落点。

### 路径规划

- 规划层使用连续物理坐标，不再采用 4 邻接 BFS。
- 静止棋子按圆形风险区建模：
  - 磁铁中心轨迹不得进入其他棋子的 `magnet_exclusion_radius`
  - 被拖棋子的中心轨迹不得与其他棋子的碰撞圆相交
- `soft_clearance` 只作为路径代价偏好，不改变硬安全半径。
- 实际路径由可见图搜索生成，候选节点围绕障碍风险区分布，边为任意安全直线。
- 终点释放方向参与规划：接近目标、终点过冲、空载回正作为一个整体选择，避免过冲方向指向邻近棋子。
- 普通走子按 `waypoints_mm + release` 执行一次连续拖动。
- 吃子流程当前按“先移走被吃子，再移动进攻方”执行；被吃子进入右侧吃子区槽位
- 当前阶段判断：高密度区域的稳定性优先靠物理规划规避磁铁误吸，而不是全局减小过冲。

## 棋盘与 XY 平台尺寸

- 平台：雕刻机框架，工作面积 `440mm × 505mm`
- 主棋盘：`9 × 10`
- 吃子区：右侧 `2 × 10`
- 点阵总数：`11 × 10`
- 棋盘纸店家标称边长约 `40cm`，但控制参数以实测值为准
- 主棋盘实测有效格区：
  - `x` 方向：`370mm`（`9` 格）
  - `y` 方向：`337mm`（`8` 格）
- 棋盘纸外形尺寸比有效格区更大，需额外预留外圈边距

## 视觉识别与补光

- 视觉链路（采集、标定、识别、JSON 导出）已迁移到同级独立仓库 `GhostVision`，相机硬件与取流参数见 `../GhostVision/docs/camera.md`
- 补光：`12V 2m 4000K` 正发光硅胶灯带
- 主仓库只接收标准化外部视觉结果，并将其投影为内部 `BoardState`

## 供电策略

- 前期优先使用各设备原装电源，降低联调复杂度
- 外设母线按 `12V` 设计，供步进电机、灯带、电磁铁使用
- 当前外设电源：`LRS-350-12`
- 控制板与 `12V` 外设母线必须共地

## 当前待继续验证的事项

- 若后续实测显示明显各向异性，再细化 `x+ / x- / y+ / y-` 四个方向的补偿值
- 在更高棋局密度下复验擦边经过和误吸情况
- 在实机上复验完整走子与吃子流程
- 在新备用棋盘与当前贴纸棋片形态下，继续复验 `x=370mm`、`y=337mm` 有效格区内的落点误差与误吸情况
